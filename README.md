# MorphoPINN: Physics-Informed Spatio-Temporal Graph Attention Network for Microplastic Drift Prediction

**A Physics-Informed Encoder-Processor-Decoder (EPD) Graph Transformer for marine microplastic forensic routing in the North Atlantic Basin.**

This repository contains the official PyTorch implementation and PyDeck simulation dashboard for the framework detailed in our journal submission (under review, 2026).

---

## Project Overview

MorphoPINN replaces classical deterministic particle tracking with a graph neural network that couples real oceanographic physics into the learning gradient. Rather than predicting static density scores, the system forecasts the continuous Eastward/Northward velocity vectors governing microplastic surface drift, evaluated across five spatial resolution protocols (15–200 km).

### Core Architectural Features

- **Spatial Graph Topology:** Transforms NCEI microplastic observation coordinates into a `G = (V, E)` point-cloud graph using a BallTree-indexed `K=10` Haversine adjacency matrix, natively respecting Earth-surface curvature.
- **Encoder-Processor-Decoder (EPD) Graph Transformer:** Three stacked GATv2 message-passing layers with a per-node `TransformerEncoderLayer` feature-space self-attention block (`d_model=64`, 4 heads). The transformer applies attention across each node's feature dimensions — not across nodes globally — correcting a prior flat-sequence attention error.
- **2D Flow Divergence Continuity Penalty:** A PyTorch `autograd` penalty on `∂v_E/∂lon + ∂v_N/∂lat`, approximating incompressible 2D flow continuity. This suppresses velocity field predictions that violate mass conservation. This is a finite-difference approximation of the divergence term, not a full Navier-Stokes solve.
- **True Physics-SMOTE:** Haversine BallTree finds `k` nearest spatial neighbors; synthetic samples are generated by interpolating `V_synthetic = V_i + α(V_j − V_i)` across features, velocity, and coordinates. A small secondary Gaussian jitter (~5 cm/s) approximates sub-grid diffusion. Applied exclusively to the training split.
- **Spatial GroupKFold Cross-Validation:** Splits on `Node_Cluster_ID` (geographic clusters), ensuring no cluster appears in both train and test simultaneously. Eliminates spatial autocorrelation leakage that invalidated all prior splits.
- **CMEMS Eulerian Integration:** `load_cmems_velocity()` loads `data/raw/cmems_velocity.nc` via xarray; `join_cmems_to_master()` spatially and temporally matches each plastic observation to the nearest CMEMS grid cell (within 24 hours). `CMEMS_U` and `CMEMS_V` are added as model features, and the physics divergence penalty is computed on the residual `(pred - CMEMS_background)` rather than raw predicted velocity.
- **Fokker-Planck Sub-grid Brownian Diffusion:** Gaussian noise `N(0, 0.05²)` added to trajectory steps during dashboard deployment to model unresolved sub-grid turbulence. This is a standard Gaussian diffusion approximation, not a full Fokker-Planck solve.
- **Apriori Meta-Mining:** Fisher's Exact Test (`p < 0.05`) validates structural associations between sampling methods (e.g., Manta Net) and marine setting categories.

## Data Sources

| Dataset | File | Role |
|---|---|---|
| NCEI Global Marine Microplastics | `data/raw/ncei_microplastic.csv` | Spatial anchor point-cloud (N ≈ 22,530 observations) |
| NOAA GDP Drifter Hourly Telemetry | `data/raw/gdp_drifter_hourly.csv` | Lagrangian velocity ground-truth (filtered: `v_res ≤ 3.0 m/s`) |

**Geographic domain:** 130°W–40°E, 0°–70°N (North Atlantic Basin only).

## Results Summary

All metrics produced with spatial GroupKFold CV (no data leakage). Best protocol is **50 km** — the natural scale of North Atlantic mesoscale eddies.

| Protocol | RMSE (m/s) | MAE (m/s) | R² | 24-hr Drift Error |
|---|---|---|---|---|
| 15 km | 0.219 | 0.214 | −4.233* | ~18.9 km |
| 50 km | 0.142 | 0.094 | 0.295 | ~12.3 km |
| 100 km | 0.171 | 0.118 | 0.157 | ~14.8 km |
| 150 km | 0.155 | 0.120 | 0.238 | ~13.4 km |
| 200 km | 0.176 | 0.132 | 0.203 | ~15.2 km |

*At 15 km, the velocity target variance approaches zero, which mathematically collapses R² into negative values — an established artifact of evaluating tightly bounded physical sequences. Operational evaluation at this scale relies on RMSE/MAE, not R².  A data sufficiency guard (`n < 50` samples → skip with warning) protects this protocol from producing misleading metrics.

**Baselines** (LSTM, CNN-LSTM, GraphSAGE, ST-GCN, ADE) use identical GroupKFold splits. Full comparison: `data/processed/final_comparison_table.csv`.

## Execution Pipeline

### 1. Environment Setup

```bash
python -m pip install -r requirements.txt
python -m pip install torch torchvision torchaudio
python -m pip install torch_geometric streamlit pydeck folium networkx scikit-learn xarray
```

### 2. ETL and Graph Construction

```bash
python src/etl_pipeline.py
```

Ingests NCEI + NOAA GDP + CMEMS, applies velocity audit (`v_res ≤ 3.0 m/s`), performs spatial-temporal join, builds Haversine graph, runs clustering for GroupKFold IDs.

### 3. Train Main EPD-PINN Model

```bash
python src/model_training.py
```

Runs all five spatial protocols (15–200 km) with GroupKFold CV. Scalers fit on training data only. SMOTE applied only inside training folds. Outputs `spatial_errors_*.csv`, `training_history_*.csv`, `st_gcn_model_*.pth`, and updated `feature_scaler_*.pkl` / `target_scaler_*.pkl`.

### 4. Train Baseline Models

```bash
python src/train_baselines.py
```

Trains LSTM, CNN-LSTM, GraphSAGE, ST-GCN, ADE with the identical GroupKFold split. Outputs `spatial_errors_{MODEL}_*.csv`.

### 5. Apriori Association Mining

```bash
python src/run_apriori.py
```

### 6. Generate Metrics

```bash
python src/calculate_metrics.py
```

### 7. Generate Comparison Table

```bash
python src/generate_comparison_table.py
```

Reads all `spatial_errors_*.csv` files for MorphoPINN and all baselines, computes RMSE / MAE / Mean 24-hr Drift Error / R² per model per protocol, and outputs `data/processed/final_comparison_table.csv`. Protocols with R² < 0 are flagged with `R2_Warning: below_mean_baseline`.

### 8. Launch Dashboard

```bash
python -m streamlit run src/dashboard.py
```

Access at `http://localhost:8501`. The spatial error bar chart reads real values from `spatial_errors_*.csv` — no hardcoded placeholders. CMEMS boundary corresponds to the actual spatial extent of the loaded `cmems_velocity.nc` product.

## Known Limitations

1. **15 km data sufficiency:** The 15 km protocol may produce fewer than 50 matched samples after the GroupKFold split. Training is automatically skipped with a warning written to `data/processed/skipped_protocols.txt`.
2. **SMOTE ghost nodes in graph topology:** `build_haversine_graph()` is called on all nodes including SMOTE synthetics, meaning synthetic points influence graph topology that real test nodes evaluate on. This is documented in the code as an open limitation requiring structural refactoring to fully resolve.
3. **2D surface manifold only:** Predictions are `(v_E, v_N)` only. Vertical velocity and sub-surface bio-fouled microplastic sinking are outside the current scope (GDP drifter ground-truth is surface-tethered).
4. **Physics-SMOTE turbulence spectrum:** The synthetic jitter uses Gaussian variance. Authentic hydro-kinetic turbulence follows Kolmogorov's −5/3 spectral power law. Transitioning to spectral turbulence generation is identified as a future improvement.
